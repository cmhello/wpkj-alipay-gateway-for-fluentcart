# WPKJ FluentCart Alipay Payment - 当面付功能

## 快速启用

### 1. 配置当面付
```
WordPress 后台 → FluentCart → 设置 → 支付方式 → Alipay
→ 勾选 "Enable Face-to-Face payment for PC/Desktop"
→ 保存
```

### 2. 支付流程
- **PC端（启用当面付）**：提交订单 → 跳转到支付页面 → 显示二维码 → 扫码支付 → 自动跳转
- **PC端（未启用）**：页面跳转支付（传统方式）
- **移动端**：WAP支付（不受影响）
- **支付宝APP内**：APP支付（不受影响）

## 技术实现

### 支付流程
1. 用户在结账页面选择支付宝并提交订单
2. 系统检测到PC端且启用了当面付
3. 调用支付宝API生成二维码
4. 重定向到专属支付页面
5. 页面显示二维码和订单信息
6. 自动轮询支付状态
7. 支付完成后跳转到订单完成页

### 后端组件
- **FaceToFacePageHandler.php** - 支付页面处理器
- **PaymentStatusChecker.php** - 支付状态查询
- **AlipayAPI::createFaceToFacePayment()** - 调用支付宝API
- **PaymentProcessor::processFaceToFacePayment()** - 处理支付逻辑
- **ClientDetector** - 智能设备检测

### 前端实现
- 专属HTML页面（内联样式和脚本）
- jQuery实现支付状态轮询
- Google Charts API生成二维码
- 响应式CSS设计

### 国际化
所有用户可见文本已支持翻译：
- 扫码支付标题
- 等待支付提示
- 成功/失败/超时消息
- 订单信息标签

## 常见问题

**Q: 移动端会显示二维码吗？**  
A: 不会，系统会自动检测设备类型，移动端继续使用WAP支付。

**Q: Test模式支持当面付吗？**  
A: 支持！Test和Live模式都完全支持当面付功能。

**Q: 二维码页面卡在"订单处理中"怎么办？**  
A: 这个问题已在v1.0.3修复。确保更新到最新版本。

**Q: 需要额外配置吗？**  
A: 不需要，只需在设置中启用即可，使用相同的支付宝密钥。

**Q: 如何关闭当面付？**  
A: 在设置中取消勾选"Enable Face-to-Face payment for PC/Desktop"。

**Q: 支付超时后怎么办？**  
A: 页面会显示超时提示，用户可以刷新页面重新生成二维码。
